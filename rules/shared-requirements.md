# Shared Requirements

Never add, remove, or modify shared requirements in `docs/shared-requirements.md` without explicit user approval.

Shared requirements are PM-owned cross-cutting rules that every PRD inherits by reference. They are NOT automatically generated by agents — they are authored by the user (or with the user's explicit approval) and represent project-wide decisions.

Agents may:
- Reference SRs in PRDs (writer)
- Verify SR compliance (reviewer)
- Suggest that a recurring pattern should become a new SR (proposal only — never write)

Agents must NOT:
- Add new SRs
- Modify existing SR definitions
- Remove SRs
- Override SRs without documenting the override in the PRD

This rule applies to all agents, skills, and conversations in this project.
