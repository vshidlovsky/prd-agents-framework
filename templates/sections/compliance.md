# Section Pack: Compliance

> **Insert into**: Contract (after Edge Cases)
> **When**: Any feature touching money movement, identity verification, transaction monitoring, or regulated data.

### Compliance / Regulatory Requirements

> **GUIDE**
> **What**: Regulatory and compliance requirements that apply to this feature.
> **Why**: Compliance is non-negotiable. Missing a verification trigger or data handling rule can have legal consequences.
> **How**:
> - Document which compliance rules apply
> - Specify what thresholds trigger verification
> - Describe how the system signals compliance requirements
> - Define the user experience when compliance is triggered
